Fibre Channel and FCoE

Fibre Channel is a high-speed network protocol that is used in SANs for data transfer between computers and storage devices. Fibre Channel-based SANs rely on specialized and dedicated hardware and create a separate infrastructure from the LAN. Fibre Channel over Ethernet (FCoE) allows Fibre Channel Protocol communication over Ethernet networks, which combines the reliability of Fibre Channel with the flexibility of Ethernet.

The following figure displays a SAN using the Fibre Channel Protocol. The two storage systems at the top connect to multiple Fibre Channel switches in the middle, which then connect to several servers at the bottom. This topology allows servers to access shared storage over a high-speed network.

Fibre Channel Protocol

Fibre Channel was initiated due to storage transport limitations, such as the parallel Small Computer Systems Interface (SCSI) bus. The Fibre Channel Protocol enables multiplexing of multiple storage commands (such as SCSI) in parallel and connects a server to multiple storage devices across large networks and geographical distances. These storage devices appear to the server's operating system as locally attached disks.

A SAN is a dedicated network for block-level storage access. It consists of specialized communication devices running the Fibre Channel Protocol, such as Fibre Channel storage arrays and servers with host bus adapters (HBAs). The main participants in a SAN are servers (initiators) and storage systems (targets). The server's HBA sends storage commands over the Fibre Channel SAN. The storage targets then process these commands.

The Fibre Channel Protocol supports upper-layer protocols like SCSI, IBM's Fiber Connection (FICON), ATM, and IP through IP over Fibre Channel (IPFC), and is used within Just a Bunch of Disks (JBOD) setups that connect disks in an arbitrated loop topology.

When transferring data in blocks, which is also known as block-based data transfer, it is important to create a secure communication infrastructure that encompasses both the communication protocol to be used and the hardware needed to support this protocol. Fibre Channel Protocol creates a secure and reliable environment by implementing strict control on the connectivity between the devices, which involves assigning different port roles (discussed in the next paragraph). Another mechanism is through control of the transmission that is exercised by the receiving side with the help of hardware memory buffers.

The Fibre Channel Protocol uses different port types to connect devices and manage traffic within a SAN. The major port types include node (N) ports, fabric (F) ports, and expansion (E) ports. The trunking expansion (TE) port is a specialized option of the E-port that connects switches to allow for SAN expansion with higher bandwidth through link aggregation.

The F Port connects a single server or storage device to a switch in a SAN. The TE_port connects switches, expands the SAN, and supports higher bandwidth using link aggregation. The N Port is an endpoint in the Fibre Channel network. The E Port connects two Fibre Channel switches together. The HBA is a hardware component that connects a host system (such as a server) to a network or storage device.

As a result, the server's operating system perceives storage array logical unit numbers (LUNs) as if they were directly connected to the server. LUNs are unique identifiers for individual storage devices or partitions within a storage array. These LUNs are visible on the Fibre Channel target across the Fibre Channel network and mimic a direct connection via a parallel SCSI bus inside the server architecture.

Fibre Channel SAN offers high-performance data center connectivity and storage for disaster recovery and business continuity solutions.

Fibre Channel storage also offers cloud storage that is used in the following tasks:

  • Data protection

  • Backup

  • Restore and replications at the block level

  • Archiving or data preservation services

  • Deduplication systems

  • Virtual tape libraries (VTLs)

Sometimes, a NAS front-end appliance is connected to the Fibre Channel SAN back-end storage subsystem to form a hybrid of NAS and SAN storage.

In Cisco data center networks, you can deploy the Fibre Channel initiator on Cisco UCS B-Series Blade Servers or Cisco UCS C-Series Rack Servers. These servers connect across the Fibre Channel SAN to the Fibre Channel target storage subsystem. The target storage subsystem can be from third-party Fibre Channel storage vendors such as Dell EMC, NetApp, IBM, Hitachi, HP, Pure Storage, Red Hat, and others.

Fibre Channel over Ethernet

In a typical data center, you would have two separate networks: one for Ethernet and one for Fibre Channel. To support various types of networks, the traditional approach uses separate redundant interface modules for each network: Ethernet network interface cards (NICs) and Fibre Channel HBAs in their servers, and redundant pairs of switches at each layer in the network architecture. Using parallel infrastructures increases capital costs, makes data center management more difficult, and decreases business flexibility.

Consolidating I/O traffic in the data center brings the Fibre Channel and Ethernet networks into a single integrated infrastructure. Merging the devices that handle Fibre Channel and Ethernet traffic provides savings in terms of equipment purchase and space, power, and cooling costs. An important pillar of this consolidated approach is FCoE.

The following figure shows three network topologies: SAN, Ethernet, and converged infrastructure (FCoE). On the left, the SAN infrastructure has two separate storage networks with their own cables. In the middle, the Ethernet infrastructure handles LAN traffic separately from SAN. On the right, the converged infrastructure (FCoE) combines SAN and Ethernet in one network; Ethernet carries both LAN and SAN traffic, which makes the system simpler and more efficient.

Note

Converged infrastructure combines computing, storage, networking, and virtualization in a unified system, which simplifies management and improves efficiency.

FCoE is an end-to-end (initiating server-to-target storage array) solution that enables Fibre Channel Protocol communication on top of converged Ethernet. FCoE creates a single converged LAN-SAN network that retains the native Fibre Channel features. The main benefit is the consolidated, centralized infrastructure.

Converged network adapters (CNAs) are components of converged infrastructure that combine the functionality of virtual Ethernet NICs, virtual Fibre Channel HBAs, FCoE switches, and storage subsystems with unified FCoE storage controllers.

FCoE technology encapsulates Fibre Channel Protocol frames within Ethernet frames, enabling the transport of Fibre Channel traffic over standard Ethernet networks. This convergence simplifies cabling and reduces the number of network adapters and switches required.

The FCoE-based storage subsystem can be attached to Cisco MDS 9000 Series switch modules, such as the following:

  • Cisco MDS 9500 Series with 10-Gbps 8-port FCoE module

  • Cisco MDS 9700 Series with 48-port 10-Gbps FCoE module

  • Cisco MDS 9220i Fabric Switch

  • Cisco Nexus 93108TC-FX Switch

  • Cisco Fabric Interconnect
